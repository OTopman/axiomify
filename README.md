## ❌ The Problem

- Duplicate types between frontend and backend
- API contracts drift over time
- Runtime validation mismatches
- Bloated `node_modules` with repetitive security middleware

---

## ✅ The Solution

Axiomify lets you define your API once and generates everything else. It acts as a single source of truth for runtime validation, type-safe handlers, and pure TypeScript frontend SDKs—all with zero external middleware dependencies.

---

## ✨ Features

- **Universal Contract Engine:** Run on Express or Fastify without changing business logic.
- **End-to-End Type Safety:** Zero `any`. Strict inference from router to frontend.
- **Zero-Dependency Security:** Declarative CORS and Native Security Headers (No `helmet` or `cors` packages required).
- **Auto-Generated SDKs:** Pure, self-contained TypeScript client proxies.
- **Engine Escape Hatch:** Easily inject custom middleware (like Multer for file uploads) only where you need it.
- **Zod-Powered Validation:** Request and Multi-Status Response validation.

## 🚀 Quick Start
### 1. The Declarative Config (```axiomify.config.ts```)
Enable enterprise security, CORS, and body limits with zero external packages.

```typescript
import { defineConfig } from 'axiomify';

export default defineConfig({
  server: 'express',
  port: 3000,
  routesDir: 'src/routes',
  cors: { origin: '[https://myapp.com](https://myapp.com)', credentials: true }, // Native CORS
  helmet: true, // Native Secure HTTP Headers
  bodyLimit: '10mb' 
});
```

### 2. Define a Route (```src/routes/users.ts```)
```typescript
import { route, z } from 'axiomify';

export default route({
  method: 'POST',
  path: '/users',
  request: {
    body: z.object({
      email: z.string().email(),
      name: z.string()
    }),
  },
  responses: {
    201: z.object({ id: z.string(), success: z.boolean() }),
    400: z.object({ error: z.string() })
  },
  handler: async ({ body }) => {
    // `body` is strictly typed. Invalid requests are blocked automatically.
    return {
      status: 201,
      data: { id: 'usr_123', success: true }
    };
  },
});
```

### 3. Consume the Typed Client (Frontend)
Axiomify generates a 100% self-contained frontend proxy.
```typeScript
import { api } from '@/lib/api'; // Auto-generated by `npx axiomify generate`

const response = await api.users.create({ 
  body: { email: 'hello@test.com', name: 'Test' } 
});

console.log(response.id);
````

## 🛠 The Engine Escape Hatch (File Uploads)

Keep the framework light. If you need heavy parsers like multer for file uploads, inject them natively via the engineSetup hook:
```typescript
import { defineConfig } from 'axiomify';
import multer from 'multer';

export default defineConfig({
  server: 'express', // or fastify
  port: 3000,
  engineSetup: (app) => {
    const upload = multer({ dest: 'uploads/' });
    app.post('/custom-upload', upload.single('avatar'), (req, res) => {
      res.json({ file: req.file });
    });
  }
});
```

```bash
npx axiomify routes    # Print routing tree
```

## 🏗 Architecture

Axiomify is built around:
- Contract Definition Layer (Zod)
- Intermediate Representation (IR)
- Runtime Adapter (Express/Fastify)
- Code Generators (OpenAPI, SDKs)
  
```
Zod → IR → Runtime + Generators
```
